浅谈最近做的几个js项目

最近做了一些js项目，都用到了node.js，完全革新了自己对JavaScript的认识，对web开发也有了新的理解。

使用场景一：Laravel页面中嵌套react

laravel mix + react + ag-grid

laravel-mix提供了简易的webpack打包方式，不需要自己安装和配置babel，一步到位。

Mix can automatically install the Babel plug-ins necessary for React support. Behind the scenes, Mix will download and include the appropriate babel-preset-react Babel plug-in.

ag-grid是一个强大的grid生成工具，支持react

总体上来说在Laraval网站里面嵌入react或者vue是相对简单的。

使用场景二：单页面html+react应用

react + webpack + babel

使用webpack生成js文件，直接在html页面加载js，css

需要使用babel来编译：

  1. babel preset env for compiling Javascript ES6 code down to ES5 (please note that babel-preset-es2015 is now deprecated)
  2. babel preset react for compiling JSX and other stuff down to Javascript

安装babel相关插件：

不要忘了添加babel配置文件.babelrc

使用babel-loader来加载：

参考教程：https://www.valentinog.com/blog/babel/

使用场景三：全栈JavaScript应用（仓库管理系统）

react-admin + loopback 3

react-admin支持多种数据包, 这个项目里面用到的是react-admin-loopback插件，需要在response header添加Content-Range字段，使用loopback-content-range插件

react-admin支持多种语言包，可以方便的设置各种版本的语言

该系统使用REST api接口，前端后端统一使用JavaScript，中间使用api来通信，这样做的好处是数据和显示的有效隔离，方便之后的页面重新设计和数据扩展。缺点是逻辑层较为模糊，没有MVC架构来的清晰。

使用场景四：Electron桌面应用

Electron + puppeteer

使用Electron桌面app来取代js脚本程序

Electron的autoUpdater有两套实现方案，官方的Squirrel方案和electron-builder的electron-updater方案

最终我选择的是electron-updater方案，它有多个好处，此处参考: https://hackernoon.com/electron-the-bad-parts-2b710c491547
